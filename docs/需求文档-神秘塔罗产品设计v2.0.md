## 一、产品概述

### 1.1 产品定位

- **产品名称：** 神秘塔罗 · 命运之门
- **一句话描述：** 一款具有仪式感、深度解读的沉浸式塔罗牌占卜应用

**核心差异化：**
- 不是"抽牌→看结果"的工具，而是一场神秘的灵性体验
- 解读不是预设文案堆砌，而是基于牌面逻辑的个性化分析
- 强调仪式感、沉浸感、个性化

### 1.2 目标用户画像

**主要用户：**
- 年龄：18-35岁
- 性别：女性占比约70%
- 特征：对神秘学感兴趣、寻求情感慰藉、喜欢仪式感、追求精神层面的指引

**典型场景：**
- 感情困惑时想要指引
- 面临重大决策需要方向
- 日常想获得一点灵性启发
- 好奇塔罗、想体验占卜

### 1.3 用户核心需求

| 需求层次 | 具体需求 | 满足方式 |
|---------|---------|---------|
| 功能需求 | 占卜、获得指引 | 牌阵系统、解读系统 |
| 体验需求 | 仪式感、神秘感 | 洗牌、选牌、翻牌的仪式流程 |
| 情感需求 | 被理解、被安抚 | 解读文案要有温度、有针对性 |
| 社交需求 | 分享、记录 | 保存记录、生成分享图 |

## 二、核心流程设计

### 2.1 完整用户旅程

**步骤1 · 首页 · 命运之门**
- 背景：深邃星空，星星缓慢闪烁
- 中央：神秘塔罗Logo，下方"开启命运之旅"按钮
- 动效：Logo微微发光呼吸感

**步骤2 · 提出你的疑问**
- 顶部："星辰正在倾听你的心声..."
- 问题类型选择：🔮 爱情 | 💼 事业 | 📚 学业 | 💰 财运 | 🏥 健康 | 👥 人际
- 问题输入框：说出你的困惑...
- 问题质量提示：好的问题 vs 不好的问题示例

**步骤3 · 选择牌阵**
- 根据问题类型推荐牌阵
- 牌阵展示：单张指引、时间流、关系牌阵、凯尔特十字
- 显示牌数、解读深度、适用场景

**步骤4 · 洗牌仪式**
- 洗牌动画（5秒）：牌堆翻滚、粒子特效
- 可跳过动画
- 文字提示："命运之牌正在为你洗牌..."

**步骤5 · 选牌仪式**
- 78张牌扇形展开（背面朝上）
- 悬停上浮、选中发光、音效反馈
- 提示："请凭直觉选择 X 张牌"

**步骤6 · 翻牌与解读**
- 牌移动到牌阵位置
- 依次翻转（间隔0.5秒），粒子特效
- 正位金色边框，逆位紫色边框
- 显示单牌解读

**步骤7 · 深度解读**
- 第一部分：元素能量分析
- 第二部分：牌位关系分析
- 第三部分：综合预言解读
- 关键指引：行动建议、注意事项、有利时机

## 三、牌阵系统详细设计

### 3.1 牌阵配置表

**牌阵1：单张指引**
- 牌数：1张
- 适用：日常指引、快速决策
- 难度：★☆☆☆☆

**牌阵2：时间流（过去-现在-未来）**
- 牌数：3张
- 适用：感情发展、事情走向、个人成长
- 难度：★★☆☆☆
- 位置关系：过去→现在→未来的影响链

**牌阵3：关系牌阵**
- 牌数：5张
- 位置：你、对方、关系现状、挑战、建议
- 适用：感情问题、人际困扰、合作关系
- 难度：★★★☆☆

**牌阵4：凯尔特十字**
- 牌数：10张
- 位置：现状、挑战、根基、过去、意识目标、潜意识、建议、外部影响、希望与恐惧、结果
- 适用：复杂问题、重大决策、全面分析
- 难度：★★★★★

### 3.2 牌阵推荐逻辑

| 问题类型 | 推荐牌阵顺序 |
|---------|-------------|
| 爱情 | 关系牌阵 > 时间流 > 单张指引 |
| 事业 | 时间流 > 凯尔特十字 > 单张指引 |
| 学业 | 时间流 > 单张指引 |
| 财运 | 凯尔特十字 > 时间流 |
| 健康 | 单张指引 > 时间流 |
| 人际 | 关系牌阵 > 时间流 |

## 四、解读系统详细设计

### 4.1 解读架构（三层）

**第一层：单牌解读**
- 牌的基础含义（正/逆）
- 关键词
- 元素/数字/符号意义
- 针对问题类型的解读（爱情/事业/学业等）

**第二层：牌位解读**
- 该位置的意义
- 牌在该位置的含义
- 与相邻位置的关系

**第三层：综合解读**
- 元素能量分析（火水风土分布）
- 牌位关系分析（影响链、互动模式）
- 针对问题的综合回答
- 行动建议

### 4.2 元素分析规则

| 元素 | 关键词 | 主导解读 |
|-----|-------|---------|
| 🔥 火 | 行动、创造、激情 | 需要主动出击，相信直觉 |
| 💧 水 | 情感、直觉、关系 | 倾听内心，关注感受 |
| 💨 风 | 思维、沟通、真相 | 理性分析，寻求真相 |
| 🌍 土 | 物质、现实、稳定 | 脚踏实地，关注实际 |

### 4.3 解读文案风格指南

**风格定位：** 神秘、温暖、有启发、不迷信

**文案原则：**
- 不说教：用"宇宙提示"代替"你应该"
- 有温度：理解用户的困惑，给予支持
- 有启发：不是告诉答案，而是提供视角
- 留有余地：塔罗是指引，不是定论

**示例对比：**

❌ 不好的写法：
```
愚者出现在过去，说明你以前很冲动。
现在的力量说明你要坚强。
未来的星星说明会有希望。
```

✅ 好的写法：
```
过去的愚者显示，你曾勇敢地开启一段新的旅程。
那种无畏的精神，至今仍在你的内心深处。

现在的力量提醒你，真正的力量来自内心的平静与坚定。
面对挑战时，温柔而坚定地相信自己。

未来的星星闪耀着希望的光芒。
黑暗终将过去，黎明正在向你走来。
```

## 五、问题引导系统

### 5.1 问题类型配置

| 类型 | 图标 | 问题模板示例 |
|-----|-----|-------------|
| 爱情 | 🔮 | 这段感情会如何发展？我们之间的缘分如何？ |
| 事业 | 💼 | 我的事业发展方向在哪里？这份工作适合我吗？ |
| 学业 | 📚 | 我的学业该如何提升？考试结果如何？ |
| 财运 | 💰 | 近期财运如何？投资需要注意什么？ |
| 健康 | 🏥 | 我的健康状况如何？需要注意什么？ |
| 人际 | 👥 | 这段关系该如何发展？如何改善人际关系？ |

### 5.2 问题质量检测

**检测规则：**
- 是非题：提示转换为开放性问题
- 太模糊：提示更具体
- 太具体（具体时间）：提示塔罗揭示趋势而非时间点

## 六、技术实现方案

### 6.1 技术栈选型

**前端：**
- 框架：Vue 3 + Composition API
- 状态管理：Pinia
- 动画：GSAP + CSS3
- UI组件：自定义玻璃态组件
- 构建工具：Vite

**后端：**
- 框架：Node.js + Express
- 解读引擎：规则引擎 + 模板引擎
- 数据存储：SQLite（MVP）→ PostgreSQL（正式）

### 6.2 项目结构

```
tarot-mystic/
├── docs/                    # 文档
├── server/                  # 后端
│   ├── routes/             # API路由
│   ├── services/           # 业务逻辑（解读引擎）
│   ├── data/               # 数据文件（78张牌、牌阵配置）
│   └── utils/              # 工具函数
├── client/                  # 前端
│   ├── src/
│   │   ├── components/     # 组件
│   │   │   ├── common/     # 通用组件
│   │   │   ├── ritual/     # 仪式流程组件
│   │   │   ├── reading/    # 解读组件
│   │   │   └── spread/     # 牌阵组件
│   │   ├── views/          # 页面视图
│   │   ├── stores/         # Pinia状态
│   │   └── assets/         # 资源
│   └── public/
└── package.json
```

## 八、验收标准

### 功能验收

| 功能 | 验收标准 |
|-----|---------|
| 问题输入 | 可输入问题，有问题类型引导，有问题质量提示 |
| 牌阵选择 | 至少3种牌阵，有推荐机制 |
| 洗牌动画 | 动画流畅，可跳过 |
| 选牌交互 | 可点击选择，有视觉反馈，选满后自动进入下一步 |
| 翻牌动画 | 依次翻转，有粒子特效 |
| 单牌解读 | 包含牌义、关键词、位置含义 |
| 元素分析 | 显示元素分布，有解读文案 |
| 综合解读 | 包含针对问题的回答，有行动建议 |

### 性能验收

- 首屏加载 < 3秒
- 洗牌动画帧率 > 30fps
- 解读生成 < 2秒
- 无明显卡顿

## 九、风险与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|-----|-------|-----|---------|
| 解读质量不稳定 | 高 | 高 | 收集反馈，迭代模板 |
| 动画性能问题 | 中 | 中 | 简化特效；提供"简洁模式" |
| 开发时间不足 | 中 | 高 | 砍功能，保核心体验 |
| 牌面版权问题 | 低 | 高 | 使用纯文字/自定义设计 |
